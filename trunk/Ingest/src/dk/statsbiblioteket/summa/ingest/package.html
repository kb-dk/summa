The Ingest package is used to build ingesting work flows.
Central for the ingest work flow is the {@link dk.statsbiblioteket.summa.ingest.Target}. A Target defines the ingest unit, and ingest rules can be configured independently for different Targets.
Ingesting is defined by 3 steps.
  • Pre-ingest filters, used to convert data from one form to another. For data enrichment a pre-ingest filter will also be the right way to proceed.
    • Multiple pre-ingest filters can be defined.
  • Ingest, used to define {@link dk.statsbiblioteket.summa.common.Record} objects and commit them to the {@link dk.statsbiblioteket.summa.MetadataStorage}
    • Implementors needs to implement a {@link dk.statsbiblioteket.summa.ingest.Ingest}, a {@link dk.statsbiblioteket.summa.ingest.Digester} and properly extend {@link dk.statsbiblioteket.summa.ingest.ParserTask}.
  • Implementors are free to extend the work flow with a post ingest step. typically tasks for post ingest would be record merging etc. An example for post ingest can be found in {@link dk.statsbiblioteket.summa.ingest.postingest}.

In general terms pre-ingest needs to produce parsable XML, ingest parses the XML and extract key attributes like recordID and record content, and post ingest examine content more closely. The ingest phase can be used to collect or mark records for the post ingest phase.

The ingest is based on XML parsing using the SXA2 standard.
